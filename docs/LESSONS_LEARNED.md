# 교훈 및 시행착오 기록

프로젝트 진행 중 발생한 문제와 그로부터 배운 교훈을 기록합니다.

## 2026-01-26: ES6 모듈 통합 시도

### 상황

- CORS 오류로 `file://` 프로토콜에서 ES6 모듈이 동작하지 않음
- AI가 즉시 해결책으로 모든 JS를 하나의 파일로 통합

### 문제점

1. **장기적 유지보수 어려움**: 파일 분리의 장점 상실
2. **향후 작업 복잡도 증가**: 수정/추가 시 하나의 큰 파일에서 작업
3. **단기적 해결책**: 당장은 되지만 앞으로 문제 발생

### 교훈

> **즉시 해결하려 하지 말고, 먼저 분석하고 여러 방안을 제시하라**

### 올바른 접근법

1. 문제 발생 → 즉시 수정 ❌
2. 문제 발생 → 분석 → 여러 방안 비교 → 사용자 선택 → 구현 ✅

### 적용된 개선사항

- `DEV_GUIDELINES.md` 문서 작성
- 이슈 분석 보고서 템플릿 도입
- `docs/issues/` 폴더에 이슈별 상세 분석 저장

## 앞으로 기억할 원칙

### 1. 구조적 변경은 신중하게

- 파일 구조, 모듈 시스템, 아키텍처 변경은 영향 범위가 큼
- 반드시 여러 방안을 비교하고 승인 후 진행

### 2. 단기 vs 장기 트레이드오프 고려

- "지금 당장 되게 하는 것"보다 "앞으로도 잘 유지되는 것"이 중요
- 빠른 해결책이 나중에 더 큰 비용을 초래할 수 있음

### 3. 문서화의 중요성

- 왜 이런 결정을 했는지 기록
- 나중에 같은 실수를 반복하지 않도록

### 4. AI와의 협업 방식

- AI에게 명확한 가이드라인 제공
- "이슈 발생 시 분석 보고서 먼저"라는 원칙 명시
- 검토 없이 진행하면 안 되는 범위 정의
