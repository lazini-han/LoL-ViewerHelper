# LoL 대회 시청 도우미 - 2단계 기능 기획서

## 1. 2단계 개요

**목표**: 1단계 MVP에서 해결하지 못한 시청자 문제점들을 해결하는 정보 제공 기능 추가

### 1단계에서 해결한 문제
- ✅ 문제 1: 챔피언 이름과 선수 닉네임을 혼동하는 문제 → 경기 화면에서 한눈에 확인 가능

### 2단계에서 해결할 문제
- 🔲 문제 2: 챔피언의 역할 특화를 몰라서 플레이어 평가를 이해 못함
- 🔲 문제 3: 한타 시 화면의 이펙트가 어떤 챔피언의 어떤 스킬인지 모름
- ✅ 문제 4: 아이템 효과를 몰라서 상황 변화를 파악 못함 → **아이템 사전 페이지 완료**
- ✅ 문제 5: 게임 오브젝트의 승패 영향을 몰라서 흥미 포인트를 놓침 → **오브젝트 사전 페이지 완료**

---

## 2. 기능별 상세 기획

### 2.1 챔피언 역할 정보 (문제 2 해결)

**목적**: 시청자가 "이 챔피언은 어떤 플레이를 잘해야 하는 캐릭터인가?"를 이해

#### 표시할 정보
- **주요 역할 태그**: 탱커, 전사, 암살자, 마법사, 원거리 딜러, 서포터
- **플레이 스타일**:
  - 라인전 강자 / 후반 강자
  - 스플릿 푸시 / 한타 특화
  - 이니시에이터 / 후속 딜러
- **간단한 설명** (1-2문장): "팀싸움에서 적 후방을 암살하는 역할"

#### UI 위치
- 경기 화면의 챔피언 아이콘 클릭 시 팝업/툴팁으로 표시
- 또는 챔피언 픽 페이지에서 챔피언 상세 패널 추가

#### 데이터 구조
```json
{
  "id": 86,
  "nameKr": "가렌",
  "roles": ["전사", "탱커"],
  "playstyle": {
    "phase": "라인전 강자",
    "teamfight": "전열 탱커",
    "specialty": "스플릿 푸시"
  },
  "description": "라인전에서 꾸준히 압박하고 한타에서는 적 후방을 위협하는 전사"
}
```

---

### 2.2 챔피언 스킬 정보 (문제 3 해결)

**목적**: 시청자가 "화면에 보이는 이펙트가 뭔지"를 빠르게 파악

#### 표시할 정보
- **스킬 이름** (한글)
- **스킬 아이콘**
- **간단한 효과 설명** (1문장)
- **시각적 특징**: "파란색 원형 범위 공격" 등

#### UI 위치
- 경기 화면에서 챔피언 클릭 시 스킬 목록 표시
- 또는 별도 "스킬 사전" 페이지

#### 데이터 구조
```json
{
  "championId": 86,
  "skills": {
    "passive": {
      "name": "인내",
      "icon": "url",
      "effect": "전투 이탈 시 체력 재생",
      "visual": "체력바 위 녹색 이펙트"
    },
    "Q": {
      "name": "결정타",
      "icon": "url",
      "effect": "이동속도 증가 후 강화 공격",
      "visual": "검에 하얀 빛 + 침묵 효과"
    }
  }
}
```

---

### 2.3 아이템 정보 (문제 4 해결)

**목적**: 시청자가 "저 아이템이 뭐고 왜 중요한지"를 이해

#### 표시할 정보
- **아이템 이름** (한글)
- **아이템 아이콘**
- **주요 효과** (1-2문장)
- **카테고리**: 공격, 방어, 기동력, 유틸리티

#### UI 위치
- 별도 "아이템 사전" 페이지
- 검색 기능 포함

#### 데이터 구조
```json
{
  "id": 3031,
  "nameKr": "무한의 대검",
  "icon": "url",
  "category": "공격",
  "effect": "치명타 확률과 치명타 피해량을 크게 증가시켜 원거리 딜러의 후반 화력을 극대화",
  "keyPoint": "원딜이 이 아이템을 완성하면 딜이 폭발적으로 증가"
}
```

---

### 2.4 게임 오브젝트 정보 (문제 5 해결)

**목적**: 시청자가 "왜 저걸 먹으려고 싸우는지"를 이해

#### 표시할 정보
- **오브젝트 이름**: 드래곤, 내셔남작, 전령 등
- **획득 효과**
- **게임 영향도** (중요도 표시)
- **등장 시점**

#### UI 위치
- 별도 "오브젝트 가이드" 페이지
- 또는 경기 화면 하단에 간단한 참조 영역

#### 데이터 구조
```json
{
  "objects": [
    {
      "id": "infernal_dragon",
      "nameKr": "화염 드래곤",
      "icon": "url",
      "effect": "공격력/주문력 증가",
      "importance": "높음",
      "spawnTime": "5분",
      "description": "화력을 올려주어 한타 승률에 직접적 영향"
    },
    {
      "id": "baron",
      "nameKr": "내셔남작",
      "icon": "url",
      "effect": "미니언 강화 버프",
      "importance": "매우 높음",
      "spawnTime": "20분",
      "description": "타워 공략이 쉬워져 게임을 끝낼 수 있는 결정적 오브젝트"
    }
  ]
}
```

## 3. 구현 우선순위

### Phase 2-1: 챔피언 역할 정보 (권장 첫 번째)
- **이유**: 경기 화면에 자연스럽게 통합 가능, 데이터 양 적음
- **예상 작업**:
  - champions.json에 역할 데이터 추가
  - 챔피언 클릭 시 툴팁 컴포넌트 구현
  - CSS 스타일링

### Phase 2-2: 게임 오브젝트 정보
- **이유**: 데이터 양이 적고, 시청자 이해도에 큰 도움
- **예상 작업**:
  - objects.json 데이터 파일 생성
  - 오브젝트 가이드 페이지 추가
  - 네비게이션에 메뉴 추가

### Phase 2-3: 아이템 정보
- **이유**: 아이템 수가 많아 데이터 준비 필요
- **예상 작업**:
  - items.json 데이터 파일 생성 (주요 아이템만)
  - 아이템 사전 페이지 추가
  - 검색/필터 기능

### Phase 2-4: 챔피언 스킬 정보
- **이유**: 데이터 양이 가장 많음 (챔피언 수 × 5스킬)
- **예상 작업**:
  - skills.json 데이터 파일 생성 (주요 챔피언만 우선)
  - 스킬 상세 팝업 컴포넌트 구현

## 4. 기술적 고려사항

### 4.1 데이터 관리
- 모든 데이터는 JSON 파일로 관리
- 버전 관리를 위해 데이터 버전 필드 포함
- Riot API 또는 Community Dragon에서 이미지 URL 활용

### 4.2 UI/UX 원칙
- **경기 화면 방해 최소화**: 팝업/툴팁은 필요할 때만 표시
- **빠른 접근**: 클릭 한 번으로 정보 확인
- **간결한 정보**: 긴 설명보다 핵심만 전달

### 4.3 확장성
- 컴포넌트 재사용 가능하도록 설계
- 데이터 구조 일관성 유지
- 새 페이지 추가 시 라우팅 확장 용이하도록

## 5. 네비게이션 구조 (2단계 완료 후)

```
┌──────────────────────────────────────────────────┐
│  LoL 시청용                                       │
├──────────────────────────────────────────────────┤
│  팀 설정 │ 챔피언 픽 │ 경기 화면 │ 📚 사전      │
└──────────────────────────────────────────────────┘

📚 사전 (드롭다운 또는 서브페이지)
├── 오브젝트 가이드
├── 아이템 사전
└── 챔피언 스킬
```

---

## 6. 데이터 수집 방법

### 자동 수집 가능
- 챔피언 기본 정보: Community Dragon API
- 아이템 아이콘: Community Dragon CDN
- 스킬 아이콘: Community Dragon CDN

### 수동 입력 필요
- 챔피언 역할/플레이스타일 설명
- 스킬 시각적 특징 설명
- 오브젝트 중요도/영향 설명
- 한글 설명 텍스트
